Finaly understand React useContext

#react #createContext #useContext #useEffect #useState #hooks

Oct.25 2023

Today I was working on google Oauth sign up function because the first time I pushed the code, I placed userEffect('to initiate google sign up button') in the wrong component causing the google sign up button only renders when the page is refresh.

I've solved the problem by truly understand how to use createContext and useContext. The reason why I need to use context is because I need to be able to access user login token to maintain user profile.

The followings are the steps:

  • Create a context file inside contexts folder, name it usercontext. User 'createContext' to create a 'UserContext', remember to export it.
  • You want the states (= user states) to live inside app.js so you can set it as 'value' and pass it to 'UserContext.Provider' and wrap it around the routes that need to access it. Be mindful that you can only wrap routes not inside route because 'UserContext.Provider' is not a route!
  • Once you have value set up, you can useContext in any component that is wrapped in the provider. Go to the component (say, sign-up.component), import useContext and setup 'useContext' to retrieve the states you setup in the app.js, do whatever setUser or things you need to change the state.

The takeaway is to understand it is to use value as a prop to use the states globally.
